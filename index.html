Looking Over Old Github Repos

Aug 12th, 2013

Setting up Octopress has given me a chance to look through some of my old Github repositories and re-assess what’s there and how to improve on them.

This is more like a wish list than an actual set of directions, and it reflects how much the world has moved since I first signed up for a github account.

  1. aadl-amazon-linky and superpatron. Both of these are interfaces to the Ann Arbor District Library catalog, which has a very accessible programming interface with things like RSS feeds for catalog searches. I’d like to go back and revisit two parts of this. One was a Greasemonkey plugin to put library results in Amazon; that should be a Chrome plugin. The second was a “wall of books” report displaying an RSS query as a page full of book cover images. Of the two, the second looks easiest to recreate.

  2. vielmetti-shelltools. This is a small collection of shell scripts that I carry around with me and put on whatever system I happen to be working on. It would be worthwhile to go back through and compare what I used then with what I use now and refresh the collection.

  3. postcard. This was a hopeful project to use various web-to-postcard APIs to generate custom postcards on the fly from the command line. Alas, several of those APIs have disappeared. The real action in this market is sending a real postcard from your iPhone or Android phone, with products like Postagram leading the way.

  4. twitter-mutual-friends. Once upon a time, the Twitter API was so simple that you could write a useful application in four lines of shell script. Now the API has improved itself, and it’s gotten a lot more complicated. Look at simple apps like latest_tweets by gianu to get an idea of what you’re up against now.

  5. mobile. Again, once upon a time, I had a Blackberry phone which had the world’s worst usable web browser. Fortunately there were enough web sites that had a minimalist presentation that I was able to put together a directory of workable sites, which I used for quite a while. Now I have an Android phone that runs Chrome as well as apps and I don’t need the list of sites quite so much; still, it’s worthwhile to have a place to collect them when I find them.

So when it’s time to write some code here are some ideas for where to start again!

Testing Octopress Rake New Post

Aug 11th, 2013

Aha, creating a new post is as easy as

A permalink will be generated from the title of the page. See the blogging basics page for more.

Blockquotes work like so:

Lincolns and Olds, Michiganders

Michael Joyce BAD:IDEA

BAD:IDEA is an online conference that Bill Doster and I organized more 25 years ago, in 1987; it’s tied up in the U of Michigan archives protected by FERPA. I need to pry it loose (since it’s my educational records, I should be able to see them).

If you are going to use a pull quote in your paragraph, just write as normal. You’ll find the one pithy piece that makes the whole thing hang together, and you’ll call it out with simple punctuation. The pull quote is styled with CSS so there’s no duplication of text and no need to write things in a funny way to make it make sense. The only slightly weird thing is that the pull quote text is not selectable, even though it looks like it should be. That’s mighty fancy CSS footwork!

Notes on Octopress

Aug 11th, 2013

Some notes while I am still new to the system.

  1. It doesn’t have comments; presumably there’s some way to wire up an external comment system of some type to handle that, like Disqus.
  2. I really wanted to start doing this by having a flexible sidebar. There is a sidebar with some default information in it (github repos, recent pinboard bookmarks) but not a flexible editable sidebar. I suppose that’s why there’s source code.
  3. I miss “edit this page” as a button; that’s the downside of not having a web-based content management system.
  4. Out of the box it looks nicer than Jekyll’s default config, and I heartily approve of that. I am sure there are other styles I can try too.
  5. I’m going to have to step up my reading skills if I’m going to understand everything behind this system, especially my Ruby reading skills.
  6. The default config’s mobile design is pretty good on my phone, better than the best I’ve been able to do so far with Typepad.
  7. I like it that github is doing the hosting because there’s not much to worry about that – I don’t have to do any more in the way of babysitting a machine just to have a blog.

So far so good, nothing too weird, everything pretty straightforward.

Installing Octopress on MacOS X 10.6.8

Aug 11th, 2013

Started to install Octopress on my Mac, but got into a problem where one of the dependencies wanted a later version of ruby (or at least a later patchlevel) than the one I had just built. That’s enough to stop me for now.

But not for very long!

This Stack Overflow post gives the syntax for installing a particular patchlevel of ruby, and so I did it.

Then it came to creating a repo on github, and I got confused; should the URL be vielmetti.github.com or vielmetti.github.io ? Because one part of the doc used one name and the in-line code example used another. I had a skew between the two, and now it looks like things are hopelessly confused (or at least confused enough that I should delete a repo and start over).

Finally, I found this Nice Cabbage post which strips the install down to its very barest minimum; that brought me past a hurdle with the above mentioned github repo.

One last issue: the system was complaining with an OS X credentials error, “error: git-credential-osxkeychain died of signal 11”. To fix that, I went to a page where I could get the latest version and updated my copy.

Since you’re reading this, it must work! Yay!
