Gitbook

Jun 27, 2017

Gitbook is an online service for writing books. It uses a version of Markdown for writing your book copy, and stores versions in a git repository. There are several template user interface designs, including one that reads like a FAQ or question and answer section, designed to be read one page at a time.

I’ve set up my own Gitbook account at http://gitbook.com/@vielmetti with a first book. Entitled A field guide to the Internet, abridged, it’s a very brief guide to some of the things you might find on the Internet if you were only looking in Ann Arbor for stuff about Ann Arbor.

I drew inspriation for the book from some of the better pages on Arborwiki and my old Vacuum weblog, both of which have had things I’ve written get page view after page view for more than a decade. It seemed like a reasonable approach to abridge both sites (which each run into the 1000s of pages) and pick out some of the good bits.

I haven’t used Gitbook enough yet to have a thorough impression of how well it works. As a site generator, it’s nicely packaged for use right out of the box, and it’s nice that it takes Markdown files because I have rather quite a lot of those to share. There have been a handful of quirky behaviors where I can’t tell if it’s a bug or just me misunderstanding how something works.

Edward Vielmetti
