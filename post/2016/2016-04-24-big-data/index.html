Big data, data analytics, and data mining

Big data is any data that you can’t analyze with the tools that are commonly known to you. For many people, that means a table bigger than what Excel can handle. What you consider “big data” depends on your capabilities and the tools and systems you have available to you. Sometimes people will be led down a weird path where problems that are straightforward to solve with one set of tools are also solvable with great difficulty with another set of tools; see Adam Drake, Command line tools can be 235x faster than your Hadoop cluster , for one illuminating instance of this.

Data analytics is making sense of “big data”. This is roughly report writing using systems that are unfamiliar to you because they can handle “big data”. As you get more proficient at these tools, you start to speak about them with less generic terms, e.g. “web analytics” specific to understanding the complex behavior of humans using web sites. “Data analytics” is a mushy term, but I guess some people are getting degrees in it, so it must be real.

Data mining is the process of drawing unjustified conclusions from big data. If you have enough data, and enough powerful but unfamiliar tools, you can typically find way too many correlations from that big data set that are only there because you are looking hard for them.

One place that data mining shows up in the world in science is the practice of “p-hacking”, or looking for strong results in a data set rather than testing a particular hypothesis. The weblog “Retraction Watch” has ample examples of this practice.
