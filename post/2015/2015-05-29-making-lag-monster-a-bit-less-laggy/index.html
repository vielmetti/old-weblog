Making Lag Monster a bit less laggy

@tallervielmetti is using my old black Macbook. It's never been an exceptionally fast machine, but it's been super laggy of late, hence the name Lag Monster.

I installed all kinds of crap on that machine to test it out, and never uninstalled a lot of it. It turns out that some judicious uninstalls seemed to help a lot to keep the load average down. Some hints to my future self if these ever re-emerge:

BBLaunchAgent. Yes, relics of my old Blackberry still linger on, and to really and truly kill them I had to edit plists. See Can't delete BBLaunchAgent.app process for the eventual diff I applied. For a more exhaustive list of Blackberry-uninstalling tools see Removing the Blackberry Desktop Software for Mac without having to re-download a 70mb installer from Blackberry.

Jenkins. Some time in the distant past I installed it, and it launched under Java and ran and did nothing for years except contribute to lag. Killed by removing the install directory and rebooting.

Chrome. Lots of Chrome stuff takes up memory. Particularly contributing to lag was some Reddit page.

At the end we're at

Processes: 77 total, 2 running, 1 stuck, 74 sleeping, 355 threads 22:40:09 Load Avg: 0.22, 0.67, 0.68 CPU usage: 17.75% user, 12.14% sys, 70.9% idle

Which is, I have to say, a lot better.
