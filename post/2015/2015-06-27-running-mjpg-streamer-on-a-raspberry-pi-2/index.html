running mjpg-streamer on a Raspberry Pi 2 with an old Logitech webcam

I followed the instructions on this post on LinuxMediaServer.net to get an old Logitech web camera running on a Raspberry Pi 2. Here are some of the changes I did from the original to support my hardware.

lsusb returns this device:

Bus 001 Device 004: ID 046d:08ae Logitech, Inc. QuickCam for Notebooks

I had to add a driver for the camera:

sudo modprobe gspca_zc3xx

The camera I have shows up as /dev/video0 and has a resolution of 320x240:

sudo ./mjpg_streamer -i "./input_uvc.so -r 320x240 -d /dev/video0" -o "./output_http.so -w www"

Existence proof, showing that it works. You should be able to recognize the Festifools t-shirt.


It's not an awesome camera, but the resolution is perfectly adequate to a small task. A next challenge, should I choose to take it on, is to wrap up all of the dependencies into a Dockerfile and do a Docker build. I'm going to wait on that until Docker 1.7.1 drops since 1.7.0 is not stable on ARM platforms. I haven't yet successfully managed to get Docker on the Pi to read from USB devices (but how hard, indeed, could it be).

UPDATE: looks like @chuckleb built a Dockerfile for this for Ubuntu at chuckleb/mjpg-streamer.

UPDATE 2: If you're using the Raspberry Pi camera, Miguel Grinberg has these build instructions for that device.
