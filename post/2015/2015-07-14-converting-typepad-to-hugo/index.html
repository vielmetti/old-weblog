14 July 2015

converting typepad to hugo

The task of the night was getting a few years worth of Typepad posts into Hugo.

My first step was to convert to Jekyll format, which I did using this typepad_to_jekyll Ruby gem:

https://github.com/koozie/typepad_to_jekyll

You then need to do a few things to get the Jekyll files exactly right for Hugo:

  1. Change the date format
  2. Rewrite the Categories
  3. Download and relink all of the images

Changing the format for “date” was a one-liner in perl with the -i (“edit in place”) flag. I didn’t get to the category rewrite or the image rescue yet. The approach I took was to simply hack on the output files, rather than change the source code to the typepad_to_jekyll program, but if you were serious about it you’d do the latter and fork it until it did the right thing.

Relinking all of the images requires opening a support ticket because Typepad does not have an easy image export. Actually image hosting is a real challenge and I don’t know exactly the right way to do it inside Hugo.

I’ll need to adjust the theme or pick a new theme so that it only shows the first n posts on a page and then paginates, and I also need to adjust it so that it shows categories. The show everything approach is unreasonable when you have 1000s of pages to show off. On the other hand, the most reasonable themes that I can find that look nice don’t have pagination turned on by default. There’s some learning curve to roll my own, maybe one that has some kind of reasonable sidebar structure.

Notes afterwards

Recreating more or less my original layout, but with dynamically loaded content. I guess that could work!
