Using Github Issues and Waffle.io as a personal kanban board

I’ve been using Github issues to track my personal todo list for quite some time now, and since November I’ve been using Waffle.io to help that process.

Waffle in this use is a view of a series of queues in my todo list. Most everything that I’m not doing now lives in the “backlog” queue, which I try to keep to less than 100 just to keep myself sane. (At more than 100 there’s usually a lot of duplicates; at less than 100 I’m probably not acknowledging that there’s something to be done). The “Ready” and “In Progress” queues hold no more than 5 tasks apiece, since I want to keep everything in that state visible on the screen. Finally the “Done” queue reflects how much work there is that actually got accomplished over the last week, and I’d love to say that was hovering at about 50 tasks a week though right now it’s only at 34.

Github issues are more than capable of handling a personal todo list - indeed it has a lot of features that if you tried to use all of them you’d quickly find that too much of your time was perfecting your lists and not enough time was spent doing things. So the challenge is coming up with a simple configuration of the system that doesn’t ignore what might be done, but that focuses on what really needs to be done.

The inspriation for all of this is Jim Benson’s “Personal Kanban”, and his observation that really the task simplifies down to two rules:

  • Limit work in progress
  • Visualize your work

The rest is merely commentary.

(Now I get to drag task 3568, “Write up use of Waffle for Vacuum blog”, into the “Done” category, which is kinetically very satisfactory.)
