Github-based kanban boards: Zenhub, Waffle, Huboard, Blossom et al

A kanban board is simply a two-dimensional view of a series of work queues. By moving a piece of work from one column to another, you update its status. This gives you a quick way to visualize your current priorities, as well as making it very clear when one or more of your queues are too full.

I've been using Github issues as a personal issue tracker for a while, using its milestones feature to track categories. This has been mostly a one-dimensional exercise of managing very long lists, or shorter lists. It's been cumbersome to use either labels or milestones to handle the quick need to say to yourself "I'm going to focus on this now" without the issue sinking back quickly into a mile-long undifferentiated list.

Enter a whole series of Github add-ons that all add a similar feature, the kanban-style column view. I'll list them here with any experience I currently have using them. My best bet is that at some point Github absorbs one of these and makes it its own, but that is tempered by the realization that Github issues really isn't changing much as a product.

Zenhub, zenhub.io. This is a Google Chrome plugin, which gives you a kanban-style view of your issues. It only works in Chrome. I tried it, but didn't see an easy way to configure which columns referred to which labels, and after a few tries gave up on it.

Waffle, waffle.io. At the moment this is what I'm using. You authenticate to it with Github, and then it displays your issues. Free (for now), they are advertising that it will be $7/mo for private repos, which is what I currently pay for all of the rest of Github.

Huboard, huboard.com. Free for open source repos or $7/mo/user for private repos. 14 day free trial.

Blossom, blossom.io. No free plan, 14 day trial, minimum is a 5 user plan at $22/mo.

Task Tub, tasktub.com. Company page does not load.

SweepBoard, sweepboard.com. Free for open source projects.

When I first started finding these sorts of things, I didn't really realize that it was a whole category of products. Most of them have some other features beyond the kanban to add to Github as a project management tool. I don't think there's a category winner, but if you are interested in visualizing your work and taking your todo list beyond the simple long list this is a useful set of tools to explore.
