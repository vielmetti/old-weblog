github on the very small screen is not much fun

Github is not much fun on a very small screen.

Octodroid does a competent job of letting you add comments to issues and scrolling through issues. It isn't so wonderful if you have gone to some trouble to categorize issues by milestone.

None of the github Android clients I have give any access to the github wiki.

Mostly the difficult part is that I had a well worked out workflow that moved open issues or if my inbox and into github, and the small screen simply does not make that a smooth process. Short of getting a bigger screen, I don't see any way to support that kind of flow on a small device.

I simply do not see enough on this little device screen to make it useful for assisting decision making about what to do next. While an omnipresent device is ideal for instant response to the top of the inbox, it is really limiting for handling overflow situations.

If I had to stick to the small screen indefinitely I would cancel some projects that are just too frustrating to keep up with. Surprisingly, blogging would stay. Pushing out commentary in an unlinked one way scroll only demands a modest typing skill and has only a minimum need for structure.
