11 October 2014

There's a reason I embraced blogging tools as soon as I found them

I'm playing on tilde.club/~emv.

It's interesting and fun to try to write a weblog when you don't have any content management system other than the Unix shell, but at some point quickly you end up moving a file on top of another file and wiping out a bunch of work, and then it gets less fun.

What I think I want to do there is write in Markdown, but somehow assemble bits of Markdown into what ends up looking like an old fashioned blog, with a sidebar that's interesting and ever-changing and a main blog part that's one post per day.

Tooling up to do that is proving interesting. How do you reliably convert markdown to HTML on the command line? It's not part of the standard Linux /usr/bin.

I'm also exercising "git" more these days.

Funtimes. And it's nice to be ~emv again.

Notes afterwards

Recreating more or less my original layout, but with dynamically loaded content. I guess that could work!
